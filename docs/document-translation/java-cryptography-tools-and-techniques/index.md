# Java Cryptography: Tools and Techniques

David Hook and Jon Eaves

本书可在 [https://leanpub.com/javacryptotoolsandtech](https://leanpub.com/javacryptotoolsandtech) 获取。

此版本发布于 2025-11-03。

## 介绍

!!! note

	“在不了解规则之前贸然违抗并不明智。”——T·S·艾略特

我们之前写的那本关于 Java、Bouncy Castle 以及它如何涉足加密学世界的书已经超过十年了。

自那时起变化很大，Bouncy Castle Java API 版本经历了两次 FIPS 140-2 认证，项目也从兼职推进逐步发展为由一个小团队全职支持的工作。随着时间戳协议、TLS，甚至最近加入的
JSSE 提供器等内容的引入，API 体系不断扩展。曾经称得上是尖端的 Java 1.5 已经到了生命周期末期，而像 SHA-1
这样的主力算法也逐渐被研究和技术所超越。即便是目前仍被认为安全的算法，其密钥长度也大幅增长，即便如此，一旦量子计算机真正到来，仍可能被宣告过时。

这本书的出版，正是为了弥补这方面的差距。在介绍 Bouncy Castle API 的更新与增强之外，我们也尽可能呈现了这些 API
所依据的原始资料与标准。之所以这样做，是因为理解这些实现背后的基础内容同样重要。对于我们大多数“在一线”的人来说，最大的问题往往是如何生成消息并与已有系统进行通信。正因为如此，了解期望实现的内容、可选方案或差异所在，对于应对那些必须处理但具体要求不明确的“黑箱”系统来说，往往能带来很大帮助。

### 关于本书的说明

如果你是初学者，并打算从本书入手，不妨先花些时间阅读附录 A。它介绍了一种名为 ASN.1 的数据描述语言，起源可追溯到 1980
年代——那是计算机领域一个充满激情的时代，特别有趣的是，当时我们只要遇到消息比所需长度多出一个字节，就会惊慌失措。ASN.1
也反映了这种“恐慌”，广泛用于定义各种加密协议中结构的编码方式。正因如此，书中到处会提及 ASN.1，对其有一些了解会使学习过程更加顺畅。

### 关于软件本身的一点说明

Bouncy Castle 项目自 1999 年起运行，并于 2000 年发布了正式版本。虽然 Bouncy Castle 慈善机构和 Crypto Workshop
的设立让团队成员得以全职投入 API
的开发，但我们从实践中深刻体会到，错误是难以避免的。无论是我们自身实施的问题、算法文档的错误、协议规范的缺陷，还是我们或其他用户的误解，都可能出现漏洞。在某些情况下，“错误”甚至可能是完全无法预见的——一旦发现攻击方式，原本被认为安全的算法或协议可能会瞬间被削弱到类似于把秘密喷涂在齐柏林飞艇侧面然后在全国大型体育赛事上空飞行的安全级别。这种情况需要我们持续保持高度警惕。

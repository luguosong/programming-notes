# 文档写作助手

编写或编辑 `docs/` 下的文档内容时，提供写作检查清单和结构化模板。

## 适用范围

- **适用**：`docs/` 下非翻译类内容的编写和编辑
- **不适用**：`docs/document-translation/` 翻译类内容（忠于原书风格）

## 写作检查清单

编写文档时，按以下清单自检：

### 结构检查

- [ ] H2/H3 标题无数字序号（如 `1.1`、`Step 1`）
- [ ] H2 划分知识维度，从上到下体现渐进逻辑
- [ ] H3 拆分具体知识点（H2 下 ≥3 个子概念时必须拆 H3）
- [ ] 标题使用描述性命名（如 `为什么需要连接池？` 而非 `概述`）
- [ ] 开头有「本文你会学到」要点列表

### 风格检查

- [ ] 每个技术决策从具体问题出发（问题驱动）
- [ ] 先类比再定义，先简单再完整
- [ ] 术语首次出现时有解释（术语零门槛）
- [ ] 代码示例有 title 属性、关键行有注释
- [ ] 错误标 `// ❌`，正确标 `// ✅`

### 排版检查

- [ ] 强调用反引号（`` ` ``），不用加粗或斜体
- [ ] 交叉引用用「」括住标题名，禁止"第X章"
- [ ] 图片用 `<figure>` 格式（参考 `.claude/rules/zensical.md`）
- [ ] Admonition 内容前有空行 + 缩进 4 格
- [ ] 列表前有空行
- [ ] 代码块 `` ``` `` 后有空格（带 title 等属性时）
- [ ] Emoji 每段最多 1 个，全文 10-20 个以内

### 导航检查（新建或移动页面时）

- [ ] `zensical.toml` nav 中已注册
- [ ] nav 标题与 front matter `title:` 完全一致
- [ ] 只有二级目录 index.md 设置 `icon:`

## 文档模板

根据文档类型选择对应模板：

**教程型**（自编笔记）：

```markdown
---
title: {标题}
---

# {标题}

**本文你会学到**：

- 知识点 1
- 知识点 2

## 为什么需要 X？

（问题驱动开头）

## 核心概念

### 概念 A

（类比 → 定义 → 示例）

### 概念 B

（类比 → 定义 → 示例）

## 基础用法

### 最小示例

（最简可运行代码）

### 常用 API

（扩展用法）

## 进阶用法

...

## 最佳实践

...
```

**参考型**（API/配置文档）：

```markdown
---
title: {标题}
---

# {标题}

## 概述

（一句话说明用途）

## 分组 A

### 组件 1

...

### 组件 2

...

## 配置参考

...

## 常见问题

...
```
